5.1 Network Security and Its Importance
‚ñº
SUBJECT VOCABULARY

Network security ‚Äî activities designed to protect a network and its data from threats such as viruses, hacker attacks, denial of service attacks, data interception and theft, and equipment failure.

Network security covers a wide range of activities that protect data from threats to its confidentiality, correctness (integrity) and availability.

C
CONFIDENTIALITY

Only authorised users can access the data they are permitted to see. Unauthorised access by criminals or competitors must be prevented. Data must not be intercepted or stolen.

e.g. A checkout employee cannot access financial records only the managing director should see.
I
INTEGRITY (Correctness)

Data must not be changed without authorisation. The network must not allow data to be altered accidentally or maliciously.

e.g. Incorrect health records or a corrupted manufacturing control system could have life-threatening consequences.
A
AVAILABILITY

Data must be accessible when it is needed. A network is useless if data cannot be accessed. Virus or Denial of Service (DoS) attacks can slow down or stop a network, delete data, allow eavesdropping, or alter data/code.

e.g. A hospital network going down means patient records are inaccessible ‚Äî potentially dangerous.
5.2 Reasons Why Security is Important
‚ñº

Here are specific examples of why network security matters. The data stored on a network could be:

üè¢ Required for Running the Organisation

A business might hold customer details, stock levels and outstanding orders on its network. If this data were lost and the business failed to fulfil orders, it could lose customer trust and potentially go bankrupt. Even schools depend on networks ‚Äî contact details, registers, timetables, medical conditions, resources for teachers ‚Äî could all become inaccessible if the network fails.

Real example: A number of businesses ‚Äî some very large ‚Äî have gone bankrupt due to network failures or data loss.
üîí Private and Confidential

Hospitals store patient details including medical conditions and prescriptions. There is a legal requirement to keep personal data secure. If a hospital's network is insecure and confidential data leaks, the hospital could be sued. Potential patients would lose trust in the hospital, which could ultimately be closed down.

GENERAL VOCABULARY

Confidential ‚Äî secret; information you would expect to be kept private.
Sue ‚Äî to take legal action against a person or organisation.

üí∞ Financially Valuable

Many types of data stored on business networks are financially valuable. Imagine a business planning a huge sale to increase revenue. If a competitor obtained the details of planned price reductions before the sale started, it could launch its own sale beforehand, undercutting the business and reducing the chance of the sale attracting new customers.

This is a form of industrial espionage ‚Äî using stolen data for commercial advantage.

5.3 Authentication and Validation
‚ñº
SUBJECT VOCABULARY

Two-factor authentication ‚Äî a security check where users must type in a code from a portable hardware device called a 'secure token' or from an SMS message sent to their mobile phone.

Access control ‚Äî decides which users have access to which data, and what they are allowed to do with it.

Authentication is the process of checking the identity of a user of a computer system or network. This is most commonly done by validating a username and password against details stored on a central server.

Authentication Methods:

1
Username + Password ‚Äî most common method. Weakness: people write passwords down, use guessable ones (pet names), or programs can try millions automatically (brute force).
2
PIN (Personal Identification Number) ‚Äî short numeric code. Faster but weaker than a complex password.
3
Fingerprint / Biometric recognition ‚Äî hard to steal or guess; increasingly common on phones and high-security systems.
4
Two-Factor Authentication (2FA) ‚Äî password PLUS a one-time code from an SMS or hardware token. Even if the password is stolen, the attacker cannot log in without the second factor.

Access Control:

Once authenticated, access control decides what the user can do:

R
Read-only ‚Äî can open and view the file contents, but cannot modify or delete.
RW
Read and Write (Modify) ‚Äî can read, alter contents and save changes.
FC
Full Control ‚Äî can also delete the file. Usually restricted to system administrators.
e.g. Employees of a shop can open a price list (read-only) but cannot accidentally or deliberately change any prices.
DID YOU KNOW?

A system administrator (sysadmin) manages and maintains the network system ‚Äî setting up users, handling security, installing software and many other tasks. Access controls are set by sysadmins according to management requirements.

5.4 Other Ways to Secure a Network
‚ñº

Authentication alone is not enough. Three other essential methods are: access control (covered above), firewalls, and physical security.

üî• Firewall

DEFINITION

A firewall is a network security system that monitors and controls data moving from one network to another. Often one of these networks is the Internet, so the firewall sits between the Internet and the local internal network.

A firewall inspects incoming and outgoing data and uses a set of rules (often called a firewall policy). These rules decide whether to allow or block data. Examples of what firewall rules can do:

‚Ä¢
Stop certain protocols from being used (e.g. block FTP to prevent data being copied to an external server)
‚Ä¢
Block data coming from or going to certain network addresses ‚Äî specific computers, servers, websites, or even entire countries
‚Ä¢
Stop hacking attempts by disallowing data that matches the pattern an attacker would use

A firewall inspects ALL data packets between the Internet and the internal network

üíª Software Firewall

Installed on individual computers (e.g. home PCs, laptops). Has default rules to protect from common threats. Less flexible ‚Äî rules are limited.

üñ•Ô∏è Hardware Firewall

Used by businesses with a LAN and Internet connection. Much more flexible in terms of rule configuration. Allows faster throughput of data. Fully customisable by the organisation.

üö™ Physical Security

SUBJECT VOCABULARY

Physical security ‚Äî controlling access to critical parts of a network using physical methods (such as locked doors) rather than software.

Malware ‚Äî short for 'malicious software'. A generic term for any software designed to disrupt the use of a computer system.

Physical security ensures that critical network parts can only be physically accessed by authorised people. The starting point is having servers located in a locked server room.

‚Ä¢
Electronic lock systems ‚Äî entry/exit times are recorded; cards/fobs can be instantly deactivated if an employee leaves or loses theirs
‚Ä¢
Burglar alarms ‚Äî detect and deter intruders outside business hours
‚Ä¢
Security tagging & locking down equipment ‚Äî prevents physical theft of hardware
‚Ä¢
CCTV ‚Äî records all access for later review
‚ö†Ô∏è Why physical security is critical: Anyone with physical access to a server can more easily bypass the authentication system or access controls. They could copy, modify or delete any data, or install malware to allow remote access whenever they want.

5.6 Contemporary Storage and Security
‚ñº

The types of storage systems available are constantly evolving. Two important contemporary types are NAS and USB flash drives.

üåê Network-Attached Storage (NAS)

DEFINITION

A NAS is a hardware device connected to a network to provide file storage for any device connected to that network. A home NAS might be a single hard drive; an organisation's NAS might consist of many drives.

A NAS device gives all network devices shared file access ‚Äî but internet exposure creates security risks

NAS devices often include access over the Internet and apps for smartphones/tablets. However, they are often designed for ease of use rather than security. Common security mistakes with NAS devices:

!
Not changing the default password ‚Äî anyone who looks up the default password online can access all stored files
!
Not updating the NAS software ‚Äî manufacturers release updates to fix security weaknesses. If not kept up to date, known vulnerabilities are published online for criminals to exploit

To minimise NAS security risks: Decide if data really needs to be internet-accessible; use complex passwords; remove all default passwords; ensure all software patches are applied.

üîå USB Flash Drives

USB drives are convenient but carry serious security risks ‚Äî encryption is the key defence

SUBJECT VOCABULARY

USB ‚Äî Universal Serial Bus. Sockets found on most modern computer systems.

Advantages: Easy to transport, relatively cheap per GB, very convenient.

Security risks:

!
Can carry large amounts of sensitive data on a small, easily lost device
!
Anyone finding a lost drive can access all its data
!
USB devices can contain malware that installs when plugged in
!
Can be used by an employee to steal data and sell it to others
Solution: Encrypted USB Drives

Encrypted USB flash drives are unreadable until a password or number code is entered on a small keypad on the body of the drive itself. This means even if the drive is lost or stolen, the data cannot be accessed without the PIN.

5.7 Cyber Attacks
‚ñº
SUBJECT VOCABULARY

Cyber attack ‚Äî any kind of malicious attack on a network-connected device.

A cyber attack is any kind of electronic attack on a computer system, server, network or other IT device. These attacks might be designed to:

A DoS/DDoS attack floods a server with fake requests ‚Äî legitimate users cannot access the service

‚Ä¢
Gain access to data contained within the system
‚Ä¢
Delete or modify information
‚Ä¢
Make the system unavailable for use (Denial of Service)
‚Ä¢
Physically damage a device connected to the network (usually by overriding safety limits)

Most cyber attacks can be classed as exploiting either human behaviour (social engineering) or technical weaknesses.

5.8 Types of Cyber Attack
‚ñº
SUBJECT VOCABULARY

Social engineering ‚Äî any kind of attack that takes advantage of how people behave and respond to certain situations.

Phishing ‚Äî attempting to get confidential information by sending a user a message that appears to come from a trustworthy person or organisation.

Shoulder surfing ‚Äî gaining access to information by watching someone enter it into a computer system.

Pharming ‚Äî directing Internet users to a fake website that looks like a real one, in order to obtain personal information such as passwords and account numbers.

üë• Social Engineering Attacks

Attacks that exploit human behaviour ‚Äî the attacker engineers a situation where the target gives away confidential information or access. Three common forms:

üé£ PHISHING

An attempt to get sensitive, confidential information (usernames, passwords, bank account or credit card details) by sending a message that appears to come from a trustworthy source.

How a phishing attack works ‚Äî step by step

How it works:

1
Victim receives an email appearing to be from a bank, payment site, auction site or social network, claiming their account has a problem
2
Victim clicks a link ‚Äî a fake website that looks and acts exactly like the real one opens
3
Victim enters login details ‚Äî these are passed directly to the attacker

Forms of phishing: email, fake websites, phone calls, instant messaging.

Why it works: People trust emails; many don't check the web address they are taken to or question whether their bank would really email them.

üëÄ SHOULDER SURFING

Gaining access to confidential information by directly observing a user ‚Äî possibly literally looking over their shoulder ‚Äî as they complete a task. Often used to get a person's username/password or PIN.

Classic example: At a cash machine, a shoulder surfer stands nearby and watches the user enter their PIN. The card is then stolen and used with the observed PIN to withdraw cash.

When it happens: Busy places where the target is concentrating on completing a task quickly and is less likely to notice someone nearby.

üåê PHARMING

Domain names (e.g. www.mybank.com) are translated into IP addresses by the Domain Name Service (DNS). If you've visited a site before, the IP address is stored in your DNS cache.

Pharming vs normal DNS resolution ‚Äî you type the right URL but end up on the wrong site!

How pharming works:

1
Malware delivered via email changes the IP address stored in your DNS cache for a legitimate site to a fake (bogus) one
2
Next time you visit that site (even by typing the correct URL), your browser connects to the fake site instead
3
Malware can also infect DNS servers themselves ‚Äî redirecting everyone who uses that server to the fake site

To prevent pharming: Check the http address matches the intended site; verify secure connection (https); check security certificate; install latest security updates; use antivirus software.

üíª Technical Weakness Attacks

ü©π UNPATCHED SOFTWARE
SUBJECT VOCABULARY

Unpatched software ‚Äî software that hasn't had the latest security updates applied to it, making it vulnerable to attack.

Software is very complex and security issues are usually only found after the software is in real-world use. Software makers provide patches (updates) to fix security issues as they are found. Problems arise because:

!
Security issues are discussed publicly online ‚Äî criminals use this knowledge to attack unpatched systems
!
Patches often have to be manually installed by a technician ‚Äî they can be forgotten about
!
Some organisations have no technician available to install patches, leaving software permanently vulnerable
üîå USB DEVICES

Any USB device can potentially be a security threat because it might contain malware that could be transferred to the system when plugged in, or it could be used to copy data to the attacker via the Internet. USB threats include:

‚Ä¢
Malware-infected drives deliberately left in car parks or public areas ‚Äî people find them and plug them in out of curiosity
‚Ä¢
Employees using personal USB drives that are unknowingly infected
‚Ä¢
Employees deliberately stealing data on USB drives to sell
üëÇ EAVESDROPPING

In computer security, eavesdropping means intercepting data being sent to/from another computer system ‚Äî reading it without actually copying or stealing it. The owner of the data might not know it has been read until it is used by the criminal.

Security weaknesses such as unpatched software or a USB device might allow malware to be installed that enables eavesdropping. Specialised hardware can also be used.

Just like a person can eavesdrop on a conversation without the speakers knowing ‚Äî eavesdropping on a network is simply reading data silently.
5.9 Protecting Against Security Weaknesses
‚ñº

There are many ways to strengthen computer systems and networks from attacks ‚Äî starting from when software is designed and written, through to deciding on what operating system to use and how network security provision will be implemented.

üèóÔ∏è Design and Implementation of Software

Security must be considered at the system and software design stage. Software and system designers need to consider:

?
What kind of authentication is needed?
?
Do different users need different levels of access to stored information?
?
Should warnings be issued before allowing potentially risky operations (copying/deleting large amounts of data)?
?
Does stored data need to be encrypted?
?
What threats will the software face? Will it be used on internet-connected computers? How do we guard against phishing, malware and hacking attempts?

üîç Code Review

SUBJECT VOCABULARY

Code vulnerability ‚Äî a computer program (code) that has been written in such a way that it creates a security issue that may be taken advantage of to gain access to the computer system or data within it.

Issues like bad programming practice can be minimised by having regular code reviews. Two main types:

üë§ Manual Review

Review by another programmer ‚Äî usually more senior and experienced. Checks for bad programming practices and code vulnerabilities. Labour intensive and therefore expensive.

ü§ñ Automated Review

Specialist software examines the code and highlights potential issues such as common vulnerabilities in the programming language or obvious bad practices. Cannot find every issue; also fairly expensive.

üß© Modular Testing

SUBJECT VOCABULARY

Modular testing ‚Äî testing each block of code as it is completed to ensure the code works as expected.

Modular testing is important to security because if small problems remain, they might be used by hackers to gain access to the system. For example, failing to validate input correctly could lead to a hacker being able to crash the program, then gaining access to the whole system, allowing valuable data to be obtained.

Ethical hacking is the branch of computer science that relates to cybersecurity and preventing cyber attacks. It is essentially 'good' hacking ‚Äî looking for weaknesses in software and systems so they can be improved.

KEY DISTINCTION

Ethical hacking = looking for weaknesses to improve the system = LEGAL (with permission).
Hacking = gaining unauthorised access to steal data or cause damage = ILLEGAL.

üß™ Penetration Testing

Penetration testing ‚Äî authorised attacks to find weaknesses before real criminals do

Penetration testing (often shortened to 'pen testing') is where the IT systems of an organisation are deliberately attacked to find any weaknesses.

‚Ä¢
Attacks are authorised by the organisation ‚Äî therefore legal
‚Ä¢
Run by employees of the organisation OR by a specialist external business contracted for this purpose
‚Ä¢
Pen tester tries to gain access to all systems ‚Äî looking for both technical weaknesses AND trying social engineering methods
‚Ä¢
After testing, a report is presented to a senior manager explaining what issues were found and their likely impact

üî¨ Commercial Analysis Tools

Software tools called commercial analysis tools or vulnerability scanners can be used to scan a system for vulnerabilities. They can be purchased or hired.

üîç Internal Scan

Scanning from within the network. Shows issues that could be exploited by a rogue employee or a hacker who managed to physically get inside the building.

üåê External Scan

Scanning from outside the network. Shows vulnerabilities a hacker could exploit remotely from outside the company network.

‚ö†Ô∏è Limitation: These tools can only identify already known vulnerabilities and must be kept up to date to be effective. Also, they are not restricted in who can purchase them ‚Äî so criminals can buy them too to find vulnerabilities to exploit.
