ğŸ”‘ Authentication & Access
Authentication Auth
The process of verifying the identity of a user trying to access a system.
e.g. Entering a username and password to log in to a school network.
Two-Factor Authentication Auth
A security check where users must provide two forms of identification â€” usually a password plus a code from an SMS or secure token.
e.g. Logging into your bank with a password AND a one-time code texted to your phone.
Access Control Auth
The method that decides which users can access which files and what they can do with them (read-only, read/write, full control).
e.g. A cashier can view but not edit the price list.
ğŸ’€ Attacks & Threats
Cyber Attack Attack
Any kind of malicious electronic attack on a computer system, server, network or IT device.
Social Engineering Attack
Attacks that exploit human behaviour â€” tricking people into giving away confidential info or access.
Types include: phishing, shoulder surfing, pharming.
Phishing Attack
Attempting to get confidential information by sending a message that appears to come from a trustworthy source.
e.g. A fake email from "your bank" asking you to log in to a copy-cat website.
Shoulder Surfing Attack
Gaining access to confidential information by directly watching someone enter it, such as looking over their shoulder at an ATM.
Pharming Attack
Redirecting Internet users to a fake website (even if they type the correct URL) by corrupting DNS records or the DNS cache.
Denial of Service (DoS) Attack
An attack on a network that attempts to prevent legitimate users from accessing its services, e.g. by flooding it with traffic.
Unpatched Software Attack
Software that hasn't had the latest security updates applied, leaving it vulnerable to known attack methods.
Eavesdropping Attack
Intercepting data being sent between computer systems â€” reading it without the sender/receiver knowing.
Hacking Attack
The act of gaining unauthorised access to a computer system and the data it contains.
Malware Attack
Short for "malicious software" â€” any software designed to disrupt, damage or gain unauthorised access to a computer system.
Code Vulnerability Attack
A weakness in software code that creates a security issue which attackers can exploit to gain access.
ğŸ›¡ï¸ Protection Methods
Firewall Protection
A network security system that monitors and controls data moving between networks, using a set of rules (firewall policy).
Can be software (home PCs) or hardware (business LANs).
Physical Security Protection
Controlling access to critical network hardware using physical methods such as locked server rooms, electronic fobs, and security cameras.
Audit Trail Protection
A chronological, tamper-proof record of activities on a computer system â€” who did what and when.
Modular Testing Protection
Testing each block/module of code as it is completed to ensure it works correctly and is free of security flaws.
Penetration Testing Protection
Deliberately attacking IT systems (with permission) to find weaknesses before criminals do. Also called "pen testing" or "ethical hacking".
â˜ï¸ Storage
Network Security Storage
Activities designed to protect a network and its data from threats such as viruses, hacker attacks, denial of service, data theft and equipment failure.
USB (Universal Serial Bus) Storage
A common connection standard found on most computers. USB flash drives are a security risk as they can carry malware or be used to steal data.
